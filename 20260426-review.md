# 專案開發回顧 - 2026-04-26

## 已完成功能
1. **Swift 解析引擎整合**：成功透過 CGO 整合 `swift-markdown`，取代原有的 `goldmark`，大幅提升解析效能與語法支援度。
2. **異步渲染架構**：實作非同步 Markdown 渲染，大檔案解析不再導致 UI 卡死。
3. **多主題系統**：支援 6 種佈景主題（含 Auto 模式），並實作自動化的 Highlight.js 樣式切換。
4. **國際化支援 (i18n)**：設定面板與 macOS 原生選單均支援多語言切換（繁中、簡中、英文、日文、韓文）。
5. **檔案關聯與跳轉**：
   - 完成 macOS 檔案關聯 (`Info.plist` + UTI)。
   - 實作智慧連結跳轉（外部網址用瀏覽器開啟、相對路徑在 App 內跳轉）。
6. **操作優化**：設定面板快捷鍵調整為 macOS 標準的 `⌘,`。

## 功能實作失敗與問題記錄
1. **檔案監控 (fsnotify)**：
   - **狀況**：實作失敗。
   - **原因**：現代編輯器多採用「原子儲存 (Atomic Save)」機制，導致原始檔案被替換，監控鏈斷裂。
   - **後續處理**：嘗試過 `fsnotify` 與 `ModTime` 輪詢，但考慮到穩定性與效能，最終決定移除此功能。
2. **應用程式圖示 (App Icon)**：
   - **狀況**：目前僅使用簡易生成的圖示。
   - **建議流程**：請使用繪圖軟體設計 1024x1024 PNG，建立 `.iconset` 資料夾並利用 `iconutil` 工具轉換為專業的 `.icns` 檔案後替換 `Resources/AppIcon.icns`。

## 技術債

## 開發回顧
---
  專業 .icns 製作指南

  要產生高品質且符合 macOS 規範的圖示，您應遵循以下步驟：

   1. 設計圖示：使用繪圖軟體（如 Sketch, Figma, 或 Adobe Illustrator）設計一張解析度為 1024x1024 的高解析度 PNG 檔案。
   2. 建立圖示集資料夾：命名為 AppIcon.iconset。
   3. 產生各尺寸圖示：將 PNG 轉換為以下尺寸（皆為 PNG）：
       * 16x16, 16x16@2x
       * 32x32, 32x32@2x
       * 128x128, 128x128@2x
       * 256x256, 256x256@2x
       * 512x512, 512x512@2x
   4. 打包：在終端機執行 iconutil -c icns AppIcon.iconset -o AppIcon.icns。
   5. 替換：將產生的 AppIcon.icns 放入 md-viewer.app/Contents/Resources/ 並覆蓋舊檔。
